推荐理由

今天选 Meta FAIR 的 Sparse Delta Memory（SDM，稀疏增量记忆）。它推进的是一条台账里从没碰过的架构主轴：线性注意力（也就是 Mamba、DeltaNet、Kairos 那一整个"高效但过目就忘"的家族）的核心命门——那块固定大小、装不下东西的记忆，第一次被廉价地扩大了几千倍。过去一个多月台账里的"长上下文"三部曲（06-10 FlashMemory、06-15 MiniMax-SA、07-09 HiLS、06-24 Unlimited OCR）全都在给标准 softmax 注意力做减法；SDM 是另外半张地图——把那个跟 Transformer 掰手腕的竞争架构家族的最大短板补上。

信号上诚实交底：HuggingFace 只有 6 赞，偏弱（当天票数榜首全是具身智能 / 视频世界模型，不在本 skill 的能力机理选片轴上）。选它靠的是血统 + 全新赛道 + 机制锐度，不是票数：作者是 Meta FAIR 旗舰阵容，其中 Hervé Jégou 和 Matthijs Douze 正是向量检索工具 FAISS 和 2019 年"Product-Key Memory（乘积键记忆）"那篇的作者，Gabriel Synnaeve 是 Code Llama 的负责人。这篇的美妙之处正在此：七年前发明"如何在海量记忆里廉价寻址"的那批人，现在把这套手艺灌进了 2024 年最强的线性 RNN 里。对思染追前沿，它值在三点：（1）补齐了他对"两大架构家族谁能赢长上下文"的完整认知；（2）它把台账反复出现的暗线"结构化的更多才有用"再钉一枚钉子（3600 倍的记忆，因为稀疏所以每字算力不变）；（3）它给"知识该放在哪"——提示词 / RAG / 权重——多加了一个可落地的坐标点（parametric memory，把知识烤进那块被学出来的初始记忆里）。

七拍故事

1. 主角：一块"永远只有巴掌大"的白板

主角是线性注意力（linear attention）/ 线性 RNN 这一类模型的"记忆"。

先把两种记法摆清楚，这是全篇的地基：

  • 标准 Transformer（softmax 注意力）的记法 = 一本越写越厚的账本。 模型每读一个新词，会为它记下一条"标签 + 内容"（术语叫 KV cache，键值缓存：key 是这个词的可检索标签，value 是它携带的内容）。回答下一个词时，模型回头扫一遍账本里的每一条，谁的标签跟当前问题最搭就多看谁。好处：过目不忘，一百万字前埋的一根针，它理论上永远能翻回去找到。代价：账本随上下文一路变厚——十万字上下文就是十万条，每生成一个新字都要重扫一遍，越长越慢、越长越占内存。成本随长度涨，这是它的原罪。
  • 线性注意力 / 线性 RNN 的记法 = 一块固定大小的白板。 它不留那本厚账本，而是把读过的所有东西压缩进一块尺寸固定的"状态矩阵"（就想成一块白板）。每读一个新词：在白板上改几笔，然后把这个词本身扔掉。回答时只看这块白板，不回头翻历史。好处：不管上下文多长，每个字的计算量和占用恒定不变——又快又省，这正是它被当成"效率前沿"、被寄望取代 Transformer 的原因。代价：白板就巴掌那么大，尺寸写死了。读到第一百万个字时，这一百万字全得挤在同一块小白板上——旧的必然被新的盖掉、糊掉。所以线性注意力会"忘"，尤其在"大海捞针"（长文档里精确找出某一个具体事实）这种任务上，远打不过 Transformer。

主角的一生，就活在这块白板的容量诅咒里。

2. 困境：想让它多记，就得让它变贵——记忆和成本被焊死在一起

有人自然会说：白板小就把白板做大点嘛。

问题在于：线性注意力每读一个字，都要把整块白板重写一遍（更新整个状态矩阵）。白板越大，每一步要动的笔画就越多，每个字的计算量就等比例涨上去。于是"能记多少"（白板面积）和"每字要算多少"（成本）被死死焊在一起：想多记 → 白板变大 → 每字变贵 → 就丢掉了线性注意力唯一的卖点（恒定廉价）。

这就是整个领域忍了好几年的两难：

廉价但健忘（线性） vs 昂贵但过目不忘（softmax）——你只能二选一。

3. 旧路：DeltaNet 让白板"会擦"，但白板还是那块白板

这条线上前人做过一次漂亮的改进，值得交代，因为 SDM 的名字（Delta Memory）就从这儿来。

最朴素的线性注意力，写白板的方式是只加不擦：每来一个"标签→内容"就往白板上叠加。麻烦是——同一个标签如果先后出现两次、内容却不同（比如"这个变量现在等于 3"后来"这个变量现在等于 7"），两次会在白板上互相打架、糊成一团。

delta rule（增量法则，Delta 就是希腊字母 Δ，"差值/改动"的意思，源头是 1980 年代的神经网络学习法则，近几年被"快权重（fast weights）"这套视角复活） 的改法是：写新内容之前，先把这个标签下的旧内容擦掉，再写新的。是一次精准编辑，不是盲目叠加——像改电子表格里的一个格子，而不是在整页上乱涂。这一下让线性注意力的"联想记忆"能力大幅变强。Gated DeltaNet（门控增量网络，2025 年底 NVIDIA × Songlin Yang，把 Mamba2 的"遗忘旋钮"和 DeltaNet 的精准编辑合到一起） 又加了一个慢慢淡忘旧记忆的旋钮（gate，门控），成了线性注意力层里公认最强的那一档，也是 SDM 直接站上去的肩膀。

但请注意：DeltaNet 会"擦"、Gated DeltaNet 会"淡忘"，它们那块白板，仍然是同一块固定大小的小白板。改的是"怎么写"，没改"能写多大"。容量天花板一寸没动。

4. 转折：白板不必小，只要每次动的地方小就行

SDM 的关键 insight 一句话——

一块大白板之所以贵，是因为你每一步都把整块重写一遍。那如果白板巨大无比（几十亿个格子），而每个字只碰其中一小撮格子呢？

那"贵"就不成立了。成本贵不贵，取决于"每个字实际动了多少格子"，而不是"白板一共多大"。 把这两件事拆开——总容量和每字动的量——两难就散了：白板可以无限大（记得多），每字只动一小撮（依然便宜）。

这就是转折点：不要压缩记忆去迁就成本，而是把"每字触碰的量"压小，让容量自由放大。

5. 解法：几十亿个格子的记忆 + 一套"乘积键"廉价寻址

SDM 把那块"单一稠密小白板"，换成一张巨大的显式记忆表（explicit memory，N 个格子，N 可以到几十亿）。机制拆开是三步：

(a) 每个字只碰一小撮格子。 来一个字，SDM 不动这几十亿格子的绝大多数，只挑出一小把——大约 写 W≈64 个 + 读 R≈64 个——把 DeltaNet 那套"先擦再写"的精准编辑只作用在这几十个被选中的格子上，其余的记忆原封不动躺着。所以每个字实际动的量 ≈ 128 个格子，跟 Gated DeltaNet 一样小，与总容量 N 无关。

(b) "从几十亿里挑出这一小撮"怎么能便宜？——靠 Product-Key Memory（乘积键记忆）。 直觉上，要在 N 个格子里选出最相关的 64 个，得把 N 个都打一遍分——N 是几十亿时贵到没法算。乘积键的巧思：不建一张 N 个键的大表，而建两张各 √N 个键的小表，把它俩两两组合（做"乘积"：A 表任一个 × B 表任一个 = 一个复合地址）。于是 √N + √N 个小键，就能张成 N 个复合地址；找最匹配的，只需在两张 √N 的小表里各挑一次，开销从 N 降到约 √N。N = 10 亿时，√N ≈ 3.16 万——瞬算。这正是 Hervé Jégou、Guillaume Lample 等人 2019 年那篇《Large Memory Layers with Product Keys》的原创招式，而 Jégou 就是这篇 SDM 的作者——他把自己七年前发明的"海量记忆廉价寻址"直接嫁接进了现代线性 RNN 的状态里。

(c) 把初始记忆也学出来（parametric memory，参数化记忆）。 记忆既然变得这么大，就可以把"还没读任何输入前、这张记忆表里预先装了什么"当成可学习的参数，在预训练里让模型往里面永久烤进一批知识——相当于给这一层内置了一本百科全书。Gated DeltaNet 那块小白板装不下这种东西（学了也没用），SDM 的大记忆装得下。

三步合起来：每字算力 = 和 Gated DeltaNet 一样（只碰 ~128 格）；总容量 = 大几千倍（8B 规模下状态约 3600 倍大）。 便宜的算力 + 巨大的容量，两难被拆开。

6. 结局：长文档记忆力跳一大截，还顺手多了一本"内置百科"——但天下没有免费午餐

在同等算力、同等参数量（iso-FLOP，控制变量到 FLOPs 都一样）的严格对比下：

  • 长上下文回忆力大涨。 在 RULER（一套标准的长上下文检索测试，考模型在 4k–131k 字的上下文里找到并用上信息） 上：1.4B 模型 Gated DeltaNet 20.0% → SDM 31.2%；8B 模型 34.2% → 50.2%。差不多补上了它与"满血 softmax 注意力"（8B 时 61.2%）之间约一半的差距——却仍保住了线性注意力那份"每字恒定成本"。而且在 8B 规模上，SDM 的预训练损失（loss）甚至压到了满血注意力之下（约 0.852 vs 0.871）。
  • 常识 / 推理也涨，靠的正是那本"内置百科"。 把初始记忆学出来（parametric memory）后：代码困惑度 NLL 0.845 → 0.822、RULER 从 28.0% → 31.2%；8B 上平均任务准确率略微反超满血注意力（约 56.84% vs 56.65%）。而同样给 Gated DeltaNet 学初始状态几乎没用（白板太小装不下）——同一个机制既修好了回忆、又给了模型一个知识库。

最反直觉、也最诚实的一处——天下没有免费午餐，代价换了个地方藏：

把状态做大 3600 倍，每字的浮点运算（FLOPs）真的一点没多（因为稀疏，绝大多数记忆没碰）。听上去像白赚。但代价不在算力，在内存硬件： 这张巨大的记忆躺在 GPU 的 HBM（高带宽内存，容量大但比片上 SRAM 慢） 里，稀疏寻址是在里面到处随机跳着读，于是GPU 的"有效算力利用率"（MFU）比稠密的 Gated DeltaNet 低了约 10 倍——即便两者 FLOPs 相同。落到真实训练：8B 的 SDM 端到端慢 1.49 倍；推理解码比 Gated DeltaNet 慢约 10%（但仍比满血注意力快 6 倍）。而且这张状态占的内存跟模型参数本身一样大，只有在超长上下文（>20 万字）才划算——SDM 8B 的状态，约等于满血注意力缓存 20.34 万个字所占的内存。

一句话结局：它没有凭空变出记忆，而是把"稀缺又慢的内存带宽"换成了"廉价又充裕的算力"——用你不缺的资源，去买你紧缺的能力。

7. 内核：便宜的秘密，从来不是"状态小"，而是"每步动得少"

思染走出门带走这一颗种子——

线性注意力被诅咒了好几年的"过目就忘"，根子从来不是"状态必须小"，而是"每一步要把整块状态重写一遍"。一旦用稀疏寻址把"每字触碰的量"和"总容量"拆开，状态就能无限大而依然廉价。"更多的记忆"从来不是问题，"没结构地更多"才是问题——结构化的更多（几十亿格子但每字只碰 128 个），既便宜又管用。

来龙去脉（倒读溯源）

第 0 层（这篇 SDM）用"乘积键稀疏寻址 + 显式大记忆"，把线性 RNN 的状态扩大几千倍而每字算力不变，补上线性注意力的长上下文短板，并顺手把初始记忆当知识库。

第 1 层（直接肩膀）——Gated DeltaNet（门控增量网络）：arXiv 2412.06464《Gated Delta Networks: Improving Mamba2 with Delta Rule》（NVIDIA × Songlin Yang，2025 底）。它把两件事捏一起：delta rule 的精准擦写（改一个格子而非乱涂）+ Mamba2 的遗忘门控（慢慢淡忘旧信息）。这是当时线性注意力层里的最强档，SDM 直接在它上面动刀。同门近亲 Gated DeltaNet-2（NVIDIA，2026-05，把 delta rule 里的"擦"和"写"解耦）。

第 2 层（两条源流的合流）SDM 的本质是两个家族的联姻——

  • (甲) 线性注意力 / 循环状态一脉Linear Attention（Katharopoulos 2020，第一次把注意力写成可循环的固定状态）→ Mamba / Mamba2（状态空间模型 SSM）→ DeltaNet（把 delta rule 搬进来，Songlin Yang 等《Parallelizing Linear Transformers with the Delta Rule》arXiv 2406.06484，NeurIPS 2024；更老的根是 Schlag 等 2021"线性 Transformer 其实是快权重程序员"、以及 Schmidhuber 1992 的快权重思想）。这一脉毕生在跟"固定状态容量天花板"较劲。
  • (乙) 稀疏显式记忆一脉Product-Key Memory（乘积键记忆），Lample、Sablayrolles、Ranzato、Denoyer、Jégou 2019 arXiv 1907.05242（NeurIPS 2019）；更老的根是 Memory Networks、Neural Turing Machine（神经图灵机）。这一脉的看家本领是"如何在几百万个记忆槽里廉价寻址"。 论文里点名"最接近"的近作是 Fast-Weight Product Key Memory（Zhao & Jones, 2026）。

SDM = 把 (乙) 的稀疏寻址，灌进 (甲) 的循环状态里。 被它往前拱一步的根问题是："线性注意力的固定小状态，真的是它换取廉价的必付代价吗？"——SDM 的回答是"不必"：把"每字算力"和"总容量"解耦，就两头都要。

对思染：模型进化 & 怎么用

这一步把前沿往哪挪了

它补齐了"长上下文之争"的另外半张地图。 你台账里的长上下文全在 softmax 一侧（06-10 FlashMemory 推理侧稀疏、06-15 MiniMax-SA 训练原生稀疏、07-09 HiLS 长度外推、06-24 Unlimited OCR 输出侧）——都是给"越长越贵"的满血注意力做减法。SDM 是 linear 一侧：那个跟 Transformer 竞争、被寄望更省的架构家族（Mamba / GLA / DeltaNet / 06-19 Kairos 用来"维持世界"的线性层就属此家），它最致命的短板"过目就忘"，第一次被廉价补上。意义：线性 / 混合架构不再天生输在长文档记忆上，作为未来高效长上下文底座的竞争力实打实抬了一档。往后重点看混合架构（少数几层满血注意力 + 多数层 SDM 式记忆层）——两家的长处第一次有希望焊在一起（这条"AR/扩散 vs 线性/softmax 走向合体"的暗线，和 07-08 Nemotron 三模式、06-26 iLLaDA 同气）。

它给台账暗线"结构化的更多才有用"又钉一枚钉子。 从 06-18 循环圈数、06-21 思考挤答案、06-25 数据调度、06-28 草稿树，到今天的 SDM——盲目地"更多"总没用，结构化地"更多"才行。SDM 就是最干净的一例："状态大 3600 倍，但稀疏寻址让每字成本不变"。

它跟你早已熟悉的"记忆"话题是两个楼层。 06-16 MRAgent（图记忆）、06-23 GateMem（记忆治理）、07-06 AutoMem（记忆即技能）都是 agent 层的外部记忆（模型之外，管文件 / 图谱）；SDM 是 架构层的内部记忆（烤进模型这一层里，可寻址、可扩容）。同一栋楼的两层，互补不重叠。

能怎么用（京小贝 & 提示词）

  1. 选长上下文底座时，多了一个"第三选项"，但要拿自己的针去验。 过去是"linear（省但忘） vs softmax（准但贵）"二选一；现在多了"linear 家族 + SDM 式记忆"——有希望在每字恒定成本下拿到接近 Transformer 的回忆力。但护栏要记牢：SDM 在 RULER 上只补了约一半差距、不是追平，且在某些"多键"检索任务上仍偏弱。所以京小贝真要上长文档 / 合规检索这种高风险精确召回，别信标称——拿你真正在乎的那类"针"（比如埋在 8 万字里的某条合规条款）自建 needle 实测，验证曲线过关再用。
  1. "我们做便宜了"这句话，永远追问：哪种资源便宜了、哪种反而更贵了。 SDM 是最教科书的例子——FLOPs 一点没多，但内存带宽利用率掉了 10 倍、真实训练慢 1.49 倍。"算力便宜 ≠ 墙上时钟便宜 ≠ 成本便宜"。这条系统直觉在台账里反复出现（07-07 MIPU 训推不一致、07-09b OmniOpt 的 SOAP 质量最高却每步慢 72 倍、06-28 JetSpec 单机最快 ≠ 线上最快）。产品拍板前，把"省了哪一种、贵了哪一种"逐项摊开，别被单一指标骗。
  1. "知识放哪"这条谱系，SDM 加了一个新坐标点。 京小贝的领域知识，可以活在：提示词（Karpathy 3.0）↔ RAG / 上下文 ↔ 烤进模型的参数化记忆（SDM 的可学习初始状态）↔ 全量微调。SDM 的 parametric memory 在这条谱上插入了一个"高容量、读取廉价、可端到端学"的新点——知识不必都塞进上下文或 RAG，也能住在一层专门的大记忆里。这跟 07-04 PAW"把模糊小函数编译进权重"是同一股思潮：智能价值从在线消耗（每次都塞上下文）向离线固化（烤进权重/记忆）迁移。
  1. 提示词直觉：linear 家族是"把旧上下文糊到一块固定状态上"，这是它历史上健忘的物理原因。 如果京小贝哪天跑在 linear / 混合底座上，别指望它完美记得 8 万字前埋的某个细节——把关键事实显式锚定、并放在它被用到的地方附近，因为回忆力虽大涨，仍是概率性的、且随任务类型波动（呼应 07-09 HiLS"关键信息别埋在池化最易漏的中段"、06-24"长输出别盯自己滚雪球的中间产物"）。
