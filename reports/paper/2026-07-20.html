推荐理由

先说没选的：今天 HF 榜首那篇 407 赞《From Pixels to States》，是"综述 + 黑神话悟空数据集"，票高是沾游戏热度，不推进机理，按护栏排除。今天 HF#2 那篇 175 赞的 LongStraw（长上下文 RL）我也没选——昨天（07-19）那次运行已经明确跳过它，理由是它自陈"只证明了跑得动、没证明训得对"（execution capacity, not complete training correctness）；我认同：它让"训长上下文"变可行，却没证明"模型因此变强"，卡在本 skill 核心判据的门槛上。

所以我去找真正把"模型能做的事"往前挪了一步的那篇，选了 RxBrain（腾讯混元，全套开源），三条理由：

  1. 它推进的是一个范式，不是一个分数——机器人/具身智能里，"会说话的模型"（看懂画面、用文字做决策）和"会脑补的模型"（预测未来画面）一直是两个脑子、貌合神离。RxBrain 把二者拧成一条：推理和想象放进同一个模型、同一条规划序列，每走一步计划都绑定一张"这步做完世界该长什么样"的想象画面。命中选片正面清单最顶上的"新架构 / 新范式"。
  2. 它给出一个真能力，而不只是可行性——它把这套东西一路接到"连续机器人动作生成"，并且没喂海量真机动作数据就能迁移到真实机器人（demo：叠袜子、挪玩具）。这是"模型能做的事"实打实往前挪了一步。
  3. 信号我诚实交底——腾讯混元出品、代码+权重全套开源（约 6.2B）、上了 HF 榜；但只有 23 赞（社区没为它欢呼，我是靠范式+血统+开源度选的，不是靠票数），而且作者自称这是"迈向具身认知基座的初步一步"、没端出碾压式 benchmark 表。它卡住了一个重要方向，但"这条路能走多远"还没被数字坐实——这点后面会反复提醒你。

七拍故事

1. 主角

一个要在真实世界里动手的具身模型——不是对着一张图聊天，而是真的去把一只袜子叠起来、把一个玩具挪个地方。以及给它造"大脑"的一群人：他们想让这颗脑子既能想清楚事，又能在脑子里把画面预演出来。

2. 困境

要在世界里行动，一个 agent 必须把"高层任务推理"和"要达到的具体物理状态"接起来——先想明白做什么、按什么顺序、有什么约束，再落到世界具体会变成什么样。可今天的模型把这件事劈成了两半，两边都缺一条腿（术语当场落地）：

  • 视觉语言模型（VLM，Vision-Language Model：看图 + 用文字理解/决策的模型）：很会看懂一个场景、用文字说出下一步该干嘛。但它的计划是纯文字的——它说"接下来把袜子拿起来"，脑子里却没有一张"拿起来之后世界长什么样"的画面。会说，不会脑补。
  • 生成式世界模型（generative world model：给一个动作、预测未来画面的模型）：很会"脑补"——你给动作，它预测出接下来的画面帧。但它本质是个视频预测器，扛不起一个多步计划的抽象逻辑（任务怎么拆、有什么约束、什么条件下走哪条分支）。会脑补，不会想事。

一个会说的哑巴画家，一个会画的糊涂逻辑家。谁都没法"一边想清楚、一边把每步落到具体画面上"。 解不了会怎样？——具身 agent 的计划要么飘在文字里落不了地，要么困在画面里没有章法，而且每学一个新技能都得喂海量真机动作数据，贵到走不远。

3. 旧路

之前大家的默认做法是把两个脑子用胶水粘起来：让一个 VLM 用文字做规划，再把计划交接给另一个单独的世界模型或动作策略去执行。两颗脑子、一道胶缝。缝会漏：

  • 文字计划和视觉预测住在两个系统里，计划并没有真正"接地"到想象出的状态上；
  • 那个执行的动作策略，得靠大量机器人动作数据才训得动。

旧路的默认前提是"推理和想象是两件事、可以拆开分头做"。 RxBrain 的转折，正是从这里劈开。

4. 转折

RxBrain 的一招：把推理和想象放进同一条序列、同一个模型里。

它把一个具身计划表示成一条"文字 token 和想象画面 token 交替"的单一序列：

  • 语言，负责搭骨架——任务分解、规划原语、约束、时间顺序、决策逻辑（"先叠左边，再叠右边；如果没对齐就重来"）；
  • 视觉想象，负责填血肉——给每一个规划步预测出对应的目标画面（把"袜子叠好一半"那张图想象出来），让每一步计划都绑定一个具体的物理状态。

语言说"下一步把左边折过去"，视觉想象当场渲染出那张"袜子被折了一半"的目标图。于是计划不再是空话，也不再是没章法的画面流——它是一句话 + 一张想象图的连续接力。

5. 解法

落地成一个统一的多模态"混合专家式 Transformer"（Mixture-of-Transformers, MoT，约 6.2B 参数）。先把 MoT 落地：不是一整块大 Transformer 硬扛所有模态，而是给不同模态（语言 / 图像 / 视频）配各自专精的 Transformer 专家，但让它们共处同一条序列、同一套注意力里——这样"推理"和"想象"能在同一个脑子里互相看见、互相影响（这正是旧路两个分离系统做不到的）。图像那一路用一个 FLUX VAE（把画面压成/还原回模型能生成的 token）来做想象画面的生成。

训练数据怎么来？一条自动流水线：拿普通的具身视频（比如有人在叠袜子），自动拆成一个个规划步，再把每一步对齐到画面里的状态变化上——就这样批量造出"文字步骤 + 想象画面"的联合监督，不用人工标注。

配套还立了个新考卷 RxBrain-Bench，专门考一件别人不考的事：模型能不能把计划表示成"文字 + 画面"联合的东西，而不是"要么只会理解、要么只会生成"的割裂两半。

6. 结局

它同时保住了理解和生成能力，并且真能产出"文字推理 + 世界状态预测 + 联合子目标规划"三者耦合的计划（模型的三大能力：具身理解与推理 / 世界状态预测（给动作预测未来帧）/ 联合子目标规划（同时吐出动作文字和目标画面））。

最反直觉、也最该记住的副发现它一路接到连续机器人动作生成，而且没有经过大规模真机动作数据预训练，就在真实机器人上跑出了不错的表现（开源 demo：umi_fold_sock 叠袜子、bridgev2_move_toy 挪玩具）。

为什么"先想象目标画面"能省掉海量动作数据？——因为想象出的那张"目标状态图"本身就是最好的中间监督。模型不必被灌几百万条动作轨迹才学会动；它只要学会"想清楚每一步该把世界变成什么样"，动作就有了明确的锚点去对齐。监督从"抄一整条动作轨迹"变成了"盯住每一步的目标状态"，数据效率就上来了。

作者自己按住了刹车这被明确称作"迈向具身认知基座模型的初步一步（an initial step）"——它端出的是一个范式和一套开源系统，不是一张碾压式 SOTA 表。它在卡位一个方向，不是在刷榜。

7. 内核

规划不该是"光用嘴说"或"光靠脑补画面"，而是两者拧成一股绳——用语言搭骨架、用想象填血肉，每一步都落到一张"接下来世界该长什么样"的具体画面上。 把"推理"和"想象"放进同一个脑子、同一条序列，模型才第一次能既想得清楚、又看得见自己要去哪；而"想象出的中间状态"顺手成了最省数据的监督信号。

来龙去脉（倒读 2–3 层）

第 1 层：它同时改两条家谱。 一条是 VLM/VLA（Vision-Language-Action，视觉语言动作模型，如 RT-2、OpenVLA：VLM 直接吐动作，但规划仍是文字/离散动作，不"想象"未来）；另一条是生成式世界模型（如本台账 06-19 讲的 Kairos、07-03 讲的 Orca：擅长预测未来帧，但不承载多步计划的抽象逻辑）。RxBrain 的态度是：这两条线各只拿到一半，得缝成一条——把 VLA 的"会想事"和世界模型的"会脑补"合进一个脑子。

第 2 层：它借的零件，来自"把视觉塞进语言大脑"这股潮。 用统一自回归 + 把图像 VAE token 化（FLUX VAE），让"理解"和"生成"住进同一个自回归主干——这条线和我们 07-16 讲的"视觉预训练当语言智能基底"、06-17 讲的京东 JoyAI 多模态是同一股水。RxBrain 的新意在接线位置：第一次把"想象下一步的画面"接到"推理下一步该干嘛"的同一条规划序列上，让语言步和想象图交替生成、彼此接地。

第 3 层：根，是认知科学几十年的老命题——"思考离不开心理预演"（mental imagery / mental simulation）。 人规划复杂动作时，会在脑子里"预演"每一步的画面。RxBrain 本质是把这个几十年的认知直觉，做成了一个可训练的损失函数：逼模型规划时同时吐出"文字步骤"和"想象画面"。这和我们 07-16 那篇"把认知科学结论第一次做成预训练损失"，是同一种把老智慧工程化的路子。

一句串起来：VLA 让模型"会想事"、世界模型让模型"会脑补"，但两条腿一直分家 → RxBrain 用"一条序列里文字与想象交替"把两条腿接到一个身体上 → 顺手发现"想象出的目标状态"能当监督，省掉海量真机动作数据 → 先把范式卡住，效果留给后人用数字填。

对思染：模型进化 & 怎么用

这一步往哪进化了。 模型进化有一条主线是"从会说，到会做"——而通往"会做"的关键，未必是更多动作数据，而是让模型学会'想象中间状态'并拿它当自己的监督信号。RxBrain 标志着规划正在从"纯语言的思维链（CoT）"升级成"语言 + 想象的双轨链"：行动前先在脑子里把每一步的结果画出来。谁能让 agent 这样"先预演再行动"，谁的 agent 就更稳、更省数据。它也顺手重新定义了"世界模型"的角色——不只是被动预测视频，而是当规划的"接地气部件"。别被"机器人"三个字挡住：这背后的认知架构（推理⊗想象）是通用的。

能怎么用（落到你的提示词和京小贝）。

  1. 一个可直接搬的规划框架——"计划 = 抽象骨架（语言）+ 落地想象（每步的目标状态）"。 京小贝做多步任务时（比如"帮我把这笔钱按稳健配置重新摆一遍"），别让模型只吐一串文字步骤；让它每一步都显式"想象/写出"执行后的目标状态——这步做完，持仓结构 / 风险敞口 / 现金比例会变成什么样。中间状态显式了，计划才可检查、可纠错（哪一步的目标状态不对，一眼就能拦下）。这就是 RxBrain"每步绑一张目标画面"在纯文本 agent 上的同构版——用"目标状态描述"代替"目标图像"。这是一条能立刻改进你提示词的手感。
  1. "省动作数据"的启发——能力迁移的瓶颈常常不是'缺演示数据'，而是'缺中间监督'。 想让模型学会一个新流程，与其狂喂完整轨迹（几百万条对话/操作），不如把关键步骤的"期望中间结果"标出来当锚点。产品上：与其攒海量完整对话去微调京小贝，不如在关键决策步标注"这一步应该达到的中间状态/结论"，用它当监督，数据效率会高得多。
  1. 一句冷静，也是最重要的产品纪律：分清"范式卡位"和"效果验收"。 RxBrain 自陈是"初步一步"、没端硬 benchmark、社区票也不高（23 赞）。它把一个重要方向（推理+想象合流）卡住了，但"这条路能走多远"还没被数字证明。看你自己产品里的新范式尝试时，盯死这条线：范式方向对 ≠ 效果已经到，别把"方向性正确"当成"已经赢了"。这也是我今天为什么放过了 175 赞的 LongStraw——票高、方向也对，但它自己都没证明"能让模型变好"。追前沿最省钱的一课，就是永远把"能跑通/方向对"和"真的更好"分成两栏记账。
