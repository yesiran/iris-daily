推荐理由

今天这篇不是"又一个把分刷高零点几"的论文，它回答的是一个当下全行业吵得最凶、却谁也没证据的根本问题：用强化学习（RL）做后训练，到底是"教会了模型新的推理能力"，还是"只是把模型本来就会的东西挑出来用得更利索"？

选它三个理由：

  1. 正中前沿争论的靶心。 2025 年那篇著名的 "Does RL Really Incentivize Reasoning Beyond the Base Model?" 说"RL 只是把基座里已有的答案 sharpen（锐化/挑出来），并没有扩展模型的能力边界"。这篇用一个能完全控制变量的实验，给出了比"是"或"否"都更细的答案——易题上确实只是锐化，难题上却真能挖出基座里"几乎沉默"的正确解。
  2. 血统够硬（弥补 HuggingFace 只有 21 赞的弱信号）。 作者里 Pavel Izmailov（前 OpenAI 推理团队）、Matus Telgarsky（做深度学习理论的知名学者）、Micah Goldblum，都是"研究模型本身机理"这一派的硬核人。这类"科学解剖"论文的特点就是当天投票不高、但半年后被反复引用——引用数当"信号"在这里失灵，血统和方向才是真信号。诚实说：它 HF 榜信号偏弱，是靠内容选进来的。
  3. 对思染是"认知底座"级的一篇。 它直接回答"模型进化靠什么"——不是玄学，是预训练 loss 几乎完美预测后训练能达到的天花板。这对京小贝换模型、评估"这个模型微调后能不能追上"，是一条产品生死线。

七拍故事

1｜主角：一群想把 RL 放到显微镜下看清楚的人

主角是"推理能力"这件事本身，和一群不甘心的研究者。今天所有前沿模型（DeepSeek-R1、o 系列那一路）变强，靠的都是同一招：RL 后训练——模型先预训练读海量文本，再用强化学习去练解题。"强化学习"这里就是：让模型自己做题，做对了给奖励、做错了不给，反复练，模型慢慢把"能拿奖励的做法"的概率调高。

问题是：这一招每天在用，但没人真正看清它对模型内部做了什么。 这群人想把它拆开看个明白。

2｜困境：两个最基本的问题，一个都答不上

后训练时代，两个问题基础到不该没答案，却真没有：

  • 问题 A预训练那一步（模型多大、读多少数据）到底怎么决定后面 RL 能拿多少回报？换句话说——花在 RL 上的算力，回报是被前面那一步锁死的吗？
  • 问题 BRL 到底对模型做了什么？是教会了它新招（原来不会，现在会了），还是只是把它本来就会但不常用的招挑出来放大？

解不了会怎样？会一直在黑箱里烧钱：不知道该把算力更多投给预训练还是投给 RL，也不知道换个更强的基座值不值。利害就是钱和方向。

3｜旧路：三条路都走不通

  • 旧路一把预训练和 RL 分开研究。各研究各的 scaling law（规模定律，就是"投入越多、能力越强"的那条经验曲线），两条线从不缝在一起，于是"预训练怎么影响 RL"这个交界处永远是空白。
  • 旧路二直接在真实大模型上做实验回答问题 B。做不了——真实模型的预训练语料是几万亿 token 的互联网大杂烩，完全不可控，你根本没法把"这个行为是预训练带来的还是 RL 带来的"归因清楚。
  • 旧路三靠嘴仗。"RL 只锐化不扩展"派 vs "RL 能教新能力"派，吵了一年，因为谁也拿不出一个能完全控制变量的干净实验。

4｜转折：把国际象棋当"受控显微镜"

作者的关键转弯：不在真实语言模型上纠缠，改用国际象棋当受控实验台。

为什么象棋是天才选择？因为它同时满足三个真实 LLM 给不了的条件——

  • 封闭可控棋局的"语料"你可以自己造、自己配比，干干净净，能精确归因。
  • 对错可验证一步棋是不是最优解，象棋引擎一秒判定，不需要人打分（这正是 RL 需要的"可验证奖励"）。
  • 可负担地跑全流程象棋模型小，作者能从 500 万参数一路跑到 10 亿参数、10 个不同大小，每个都完整走一遍"预训练 → 监督微调 → RL"，把每个变量单独拧动看结果。这在真实 LLM 上贵到做不起。

一句话：他们造了一个能把"预训练→RL"整条流水线放到显微镜下、逐个变量拧的沙盘。

5｜解法：联合 scaling law + 按难度解剖 RL

具体两件事：

（1）建一条"联合 scaling law"，把预训练和 RL 缝到一根公式里。先解释几个词，全落地：

  • 预训练 loss（pretraining loss）：模型读文本时"预测下一步"的平均错误程度，越低说明模型把这个领域学得越透。它是衡量"基座有多好"的单一数字。
  • pass@1 / pass@16给模型一道题，只让它答 1 次、答对的比例叫 pass@1（衡量"一发入魂"的准头）；让它答 16 次、只要有 1 次对就算对，叫 pass@16（衡量"这题它到底会不会、多试几次能不能蒙对"，反映的是能力的"广度/多样性"）。这两个指标的分野是本文最锋利的刀。
  • compute-optimal allocation（算力最优分配）：总算力固定，该分多少给预训练、多少给 RL 才最划算。

（2）按题目难度把 RL 的动作解剖成三类（这是本文最狠的地方）。他们把 RL 训练前后模型的变化拆开，看每一步棋的概率被 RL 怎么改了：

  • "放大已会的正确招"这招 SFT（监督微调后的）模型本来就偏爱，RL 只是把它的概率调得更高——这就是"锐化/sharpen"。
  • "挖出几乎沉默的正确招"这招在 RL 之前概率低到几乎为零，RL 把它从"沉默"里捞了出来——这才是"发现新能力"。
  • "强化错招"RL 也会把某些错误走法的概率调高——这是副作用。

用这两件工具，他们就能同时回答问题 A（scaling law）和问题 B（三类动作的占比）。

6｜结局：三个能复述的发现 + 一个反直觉炸弹

发现一：预训练 loss 几乎"算命式"地预测后训练能达到的高度。 预训练 loss 越低，同样的 RL 算力下最终得分越高，两者相关性 ρ（斯皮尔曼相关系数，衡量两个量"同涨同跌"有多齐，1 是完美同步）高达 −0.93 到 −0.99（负号是因为 loss 越低分越高，几乎是一条完美反比直线）。而且 RL 的进步速度随预训练 token 量近似线性上涨（相关性 r = +0.84）。翻译成人话：基座练得越透，RL 的杠杆越长。

发现二：算力该怎么分——RL 的份额随总预算增大而上升。 从约 20%（5000 万参数规模）涨到约 28%（6.8 亿参数规模）。预算越大，越值得多分给 RL。但——

发现三（最关键）：RL"强烈受初始化限制"。 原文："RL is strongly initialization-limited"——在总预算相同的情况下，过早开始 RL（省下预训练、把算力挪给 RL）换来的那点额外 RL 算力，补不回一个更弱的基座。基座弱，RL 再猛也追不回来。

问题 B 的答案（按难度分裂）

  • 易题约 80% 是"放大已会的正确招"——RL 基本只是锐化，印证了"RL 只 sharpen"那派。
  • 难题约 35% 是"挖出几乎沉默的正确招"（真发现），但同时约 30% 是"强化错招"（真副作用）。所以在难题上，RL 确实能教出基座里近乎没有的解法——但也会同样自信地把错招练得更牢。
  • 一个照妖镜指标RL 让 pass@1（准头）全面上涨，但 pass@16（多试几次会不会）几乎不动、大模型上还会退化。这解释了为什么"多采样投票"救不了难题——RL 没有真正拓宽能力的"广度"，只是把已有分布压得更尖。

math 上复现换成一个 10 亿参数、读了 2000 亿 token 数学文本的真语言模型，同样的规律再现——预训练读得越久，后训练天花板越高、RL 起效越快。说明这不是象棋特例，是通用规律。

反直觉炸弹大家常假设"RL 后的策略 = 把 SFT 策略整体升温（π_RL ∝ π_SFT 的 α 次方，一个均匀的温度调节）"。作者一测，这个假设只能勉强拟合——说明 RL 干的不是"整体均匀升温"，而是因题、因局面而异的定点重塑：它在不同难度、不同棋局上做的事完全不一样。这一刀直接否掉了"RL=统一调温"的朴素直觉。

7｜内核：基座是矿，RL 是矿工

带走这一句就够了：RL 不是一台能无中生有的发电机，它是一个矿工。

预训练把"潜在的推理能力"像矿脉一样埋进模型；RL 是去开采。富矿（好基座）才挖得出金子——这就是"初始化受限"。 但在难题上，好矿工确实能挖出矿脉里那些"几乎沉默、你以为没有"的金子（真发现），代价是也会挖出一堆假金子（强化错招）。所以答案不是非黑即白的"RL 只 sharpen"或"RL 教新能力"，而是：易题上它锐化，难题上它开采+夹带私货，而整座矿山的储量，在预训练那一刻就定死了。

来龙去脉（倒读溯源 3 层）

第 1 层——它直接站在谁肩上： 用"国际象棋当受控实验台研究后训练推理"，本文不是第一个。前面有 How Reasoning Evolves from Post-Training Data: An Empirical Study Using Chess、Can LLMs Develop Strategic Reasoning? Chess，以及研究"预训练/中训练/RL 三者相互作用"的 On the Interplay of Pre-Training, Mid-Training, and RL。本文的独创，是第一次把预训练和 RL 缝进同一条 scaling law，并把 RL 的动作按难度定量解剖。

第 2 层——它在回答谁的争论： 上游是 2025 年那场大辩论——Does RL Really Incentivize Reasoning Capacity Beyond the Base Model?（Yue 等）用"RL 后 pass@k 不涨"论证"RL 只是把基座里已有的解锐化出来，没有扩展能力边界"。本文用可控实验给了更精细的裁决：pass@16 确实几乎不动（支持"锐化"派）；但把动作按难度拆开后，难题上 RL 又确实挖出了近乎沉默的正确解（打了"纯锐化"派一巴掌）。 两派其实都对了一半。

第 3 层——根在哪： 再往下，一条根是 RLVR（Reinforcement Learning with Verifiable Rewards，用"可自动判对错的奖励"做强化学习）——DeepSeek-R1 那条把推理能力做起来的主线；另一条根是 Chinchilla scaling law（2022 年那条"模型多大、数据多少最划算"的预训练规模定律）。本文的历史意义，是把这两条从不相交的 scaling 传统焊成一条联合定律——预训练的地基定律，第一次被接到了后训练的能力天花板上。

对思染：模型进化 & 怎么用

这一步往哪进化了

后训练时代最大的幻觉是"用 RL 就能救一个弱基座"。这篇给了钉死的定量证据：post-RL 的天花板由预训练 loss 锁死，RL 是有杠杆的放大器，不是无中生有的发电机。 前沿的关注点因此在挪——从"给 RL 堆更多算力"转向"预训练和 RL 的联合算力分配"，而且随着总预算变大，该分给 RL 的比例是上升的（20%→28%）。这是一条"从堆量到配比"的成熟化信号。

怎么用（提示词 & 京小贝产品判断）

  1. 选基座 = 选天花板，别信"微调后能追上"。 京小贝换模型、评估一个更便宜的小模型能不能通过微调/RL 追上大模型时，"初始化受限"这条定律说：弱基座的后训练天花板是钉死的，再多后训练也追不回。 评估时要看 base 的预训练质量（数据量、领域 loss），而不是只看厂商给的"RL 后 demo"——demo 好看，可能只是易题上的锐化，难题上的真实能力仍被基座封顶。
  1. 难任务必须配"验证/纠错"，不能只靠"让它想久一点"。 本文的机制说：难题上 RL（乃至推理时的"多想"）既会挖出沉默的正确解，也会自信地强化错招。对应到提示词——落在模型"几乎会但没自信"区间的任务，加 think / self-check 的引导能把沉默的正确解逼出来；但真正的难题上它也会把错招讲得同样理直气壮，所以难任务的产品设计必须外挂验证器/纠错环，"想久一点"本身不产生正确性。
  1. 别指望"多采样投票"救难题。 pass@1 涨、pass@16 不动 = RL 只压尖了分布、没拓宽能力广度。产品里"同一问题多采几条、投票选最优"这招，在模型基座能力之外的难题上收益有限；真要抬高难题上限，得回到基座或数据，而不是采样策略。
  1. 调 temperature / 加思考预算不是线性的——A/B 要分难度桶看。 "RL≠均匀升温"意味着同一个参数在易题和难题上的效果天差地别。京小贝做提示词/推理配置的 A/B 时，只看平均分会被易题的收益掩盖难题的退化——要拆难度桶分别看。这一条和 [[2026-06-22--ufp4-fp4-pretraining-shrinkage-bias]]（"看系统性漂移不看平均分"）、[[2026-07-07--mipu-inference-policy-mirage]]（推理策略才是真目标）是同一个方法论家族：别被平均数骗，去看分层的、系统性的结构。
