推荐理由

今天选 SEED，因为它正面解决 Agent 训练里最硬的那道坎：奖励太稀。

一个 AI Agent 做一件长任务——比如"在网店里帮我找到符合这些条件的商品并下单"——要连着走十几步。用现在主流的做法（强化学习）训练它，通常只在整件事做完时给一个分：成了给 1、没成给 0。可这一个分没法告诉模型：这十几步里，到底是哪一步走对了、哪一步走砸了。模型只知道"整体考砸了"，却不知道错在第几题。这就是这篇论文起名的"监督落差"（supervision gap，"批改的粒度"和"下棋的粒度"对不上：批改只到整盘，落子却是一步一步）。

SEED 的解法很漂亮：让模型自己给自己现造密集的批改。一局走完之后，你其实已经"事后诸葛亮"了——你知道这局发生了什么。那就让模型回头看自己刚走完的这一整局，用大白话总结出一条"技能"（比如"下单前必须先核对尺寸"），然后衡量这条技能会把模型自己的下一步动作往哪个方向拉多少，把这个"拉动量"当成逐字的训练信号，一点点烤进模型的权重里。

命中的质量信号：HuggingFace 每日论文榜当日第 2 名、81 个赞（当天能力/Agent 训练这条赛道里票数最高的一篇，压在它上面的只有一个开源视频理解模型，不在同一赛道）；作者来自清华、浙大、港中文、南洋理工、同济五家一线机构联手；在 Qwen2.5 系列基座上，全线超过当前最常用的 Agent 强化学习方法 GRPO（ALFWorld +16.8 分、WebShop 成功率 +15.6 分、搜索问答 +9.3 分）。

对思染追前沿的价值有三层：① 它是"能力提升型"的训练方法，正好踩在你做 Agent 产品（京小贝的多步流程）的正中心；② 它带了一颗直接砸向"提示词工程"的反直觉炸弹——同一条经验，写进提示词里，竟然不如烤进权重里管用（下文详解，这决定了"提示词的天花板到底在哪"）；③ 它把"让模型消化自己的经验、自造监督信号"这条自我进化主线又往前推了一步，能和你之前读过的 [[2026-06-09--openskill]]、[[2026-07-06--automem]] 串成一条线。

七拍故事

1. 主角

主角是一个被当成"Agent"来训练的大模型——不是让它答一道题就完事，而是让它在一个环境里连着做很多步：观察 → 决定动作 → 环境给反馈 → 再观察 → 再动作……一直走到任务完成或失败。论文里的舞台有四个：ALFWorld（在一间虚拟房间里按指令做家务，比如"把苹果放进冰箱"，要开柜子、找东西、拿、放，一步步来）、WebShop（在模拟网店里按需求买对商品）、搜索问答（要多次调用搜索、把查到的东西串起来回答）、还有两个需要看图的任务（推箱子游戏 Sokoban、算点数 EZPoints）。基座模型是 Qwen2.5-3B-Instruct（阿里通义千问的 30 亿参数版本）和更大的 70 亿版本。

2. 困境

训练这种 Agent，现在最通行的办法是强化学习——一句话讲清它的机制：让模型自己去环境里试很多次，试得好就给奖励、试得差就不给，模型顺着"哪种做法拿到的奖励高"这个方向，慢慢把自己的行为调过去。

问题出在奖励是怎么发的。因为一件长任务只有做完了才知道成没成，所以奖励只在最后发一次：整局成功给 1、失败给 0。可一局有十几步、几百个字（每个动作都是模型吐出的一串字）。这一个总分，摊不到具体哪一步、哪个字上。 模型收到"这局 0 分"，却完全不知道：是第三步开错了柜子？还是第八步东西拿反了？还是最后一步下单信息填错了？

这就是"稀疏奖励"下的"功劳/过错分不清"问题（credit assignment，强化学习几十年的老伤口）：反馈只在终点给一个笼统的分，中间每一步该记功还是该记过，无从知晓。结果就是训练又慢又飘——模型要靠海量的试错，去猜哪些中间动作是好的。解不了这个，Agent 就很难在长任务上稳定变强。

3. 旧路

之前大家怎么补这个"批改太粗"的窟窿？三条老路，各有各的堵点：

  • 老路一：就用稀疏奖励硬扛（GRPO 这一类）。GRPO 是当前最常用的 Agent 强化学习方法（"分组相对策略优化"——同一道题让模型试好几遍，谁比这几遍的平均分高就往谁的方向调）。它简单、稳，但批改粒度还是只到整局，中间步骤依旧一片黑。堵点：监督太稀，样本效率低。
  • 老路二：另训一个"过程打分器"（process reward model）。专门再养一个模型，给中间每一步单独打分，把批改做细。堵点：这个打分器得额外花大代价去训、还容易被钻空子（模型学会讨好打分器而不是真把事做好），大规模用起来又贵又脆。
  • 老路三：让模型把经验写成"技能笔记"，塞进提示词里（Reflexion / ExpeL / Voyager 这一路）。失败之后让模型用大白话反思一句"下次记得先核对尺寸"，把这句话攒进一个笔记本，下次做任务时贴在提示词开头当小抄。堵点有两个：一是这些笔记只待在提示词里（临时贴的便签），没有真正改变模型本身；二是笔记是攒死的、不更新的——模型自己在变强，可笔记还停在几轮前那个笨版本的水平，越用越过时。

4. 转折

作者转过的这个弯，关键在两句话：

第一句：批改不用等别人来给，模型自己就能造——因为一局走完之后，它是"事后诸葛亮"。任务当下走的时候两眼一抹黑，可一旦这局走完了，成败已定、过程俱在，这时候回头看，"哪几步是关键、哪步埋了雷"是看得出来的。这叫"事后（hindsight）"视角。那就让模型回看自己刚走完的这一整局，用大白话提炼出一条技能：可能是一段可复用的套路（"先查库存再下单"）、一个决定成败的关键观察（"这个页面的确认按钮在右下角"）、或一条避坑规则（"别在没核对尺寸时点购买"）。

第二句、也是真正的巧劲：别把这条技能塞进提示词，而是量出"它把模型的动作往哪拉、拉多少"，用这个"拉动量"当逐字的训练信号，把技能烤进权重。这一步是 SEED 和老路三的分水岭——老路三把技能贴在提示词上（便签，随时会撕掉、不进模型），SEED 把技能变成一把尺子去改写模型本身。

5. 解法

具体怎么做，分两幕，术语我逐个拆开：

第一幕：先教会模型"当自己的教练"（Hindsight Skill SFT，事后技能微调）。 先用基座模型在环境里跑出 1,440 局练习（180 个任务，每个任务试 8 遍）。请一个外部的强模型（GLM-5.2）当"总结老师"，读每一局走完的完整过程，写出对应的那条技能。然后拿这些"（这一局过程 → 该记住的技能）"配对，把模型微调 3 轮（微调＝拿现成例子做标准的"看着答案照抄"训练，让模型学会：给我一局过程，我能自己总结出该记的技能）。这一幕结束，模型就具备了"回看自己、提炼技能"的本事——它成了自己的教练。

第二幕：让模型边打边自我进化（Self-Evolving On-Policy Distillation，自我进化的在线蒸馏）。先把"蒸馏"这个词落地：蒸馏原本指"让一个学生模型去模仿一个更强老师模型的输出概率"——不是照抄答案，而是照抄"老师觉得每个选项有多可能"的那份倾向。SEED 的特别之处是：这里没有外部老师，学生和老师是同一个模型的两副面孔。

它是这么转起来的。模型一边正常做任务、一边给自己刚走的局现写技能。然后对它吐出的每一个动作字，算两个概率：

  • "带技能"的那副面孔（teacher / 老师端）：假如把刚写的技能也放进上下文，模型给这个动作打的概率 ℓ^skill；
  • "不带技能"的那副面孔（student / 学生端）：什么都不加，模型原本给这个动作打的概率 ℓ^θ。

两者的差 Δ = ℓ^skill − ℓ^θ，就是"这条技能把这个动作往上抬/往下压了多少"。这个差被"掐断反向传播"（detached，只当成一个固定的目标数值、不让梯度从老师端漏回去，免得模型作弊地去改老师而不是改自己），然后当成逐字的蒸馏目标：训练"不带技能"的学生端，朝着"带技能"的老师端所偏好的方向挪。再加一道"信心闸门" g = σ(β·Δ)（σ 是把任意数压到 0~1 之间的挤压函数；技能把动作拉得越明显，这一字的信号就越被看重），把没把握的字过滤掉。

最后，这份"逐字蒸馏信号"和原本的稀疏奖励强化学习并肩用：总损失 L_SEED = L_RL + λ·L_OPD（前一项是老办法给的整局奖励，后一项是新造的逐字密集指点，λ 调两者配比）。

"自我进化"这四个字的落点技能不是攒死的——每一轮都用模型当前这个更强版本、刚跑出来的新鲜局，重新现写。模型进步了，它给自己写的技能也跟着升级，永远贴着当前水平走，不会像老路三那样过时。

6. 结局

解出来，世界变成这样（基座 Qwen2.5-3B，跟最常用的 GRPO 比）：

  • ALFWorld（虚拟家务）91.8% 成功，GRPO 只有 75.0%，高 16.8 分；
  • WebShop（模拟网购）成功率 78.9% vs 63.3%（高 15.6 分），任务完成分 88.5 vs 79.8；
  • 搜索问答45.7% vs 36.4%（高 9.3 分）；
  • 要看图的任务（推箱子 / 算点数）：平均 91.0% 成功；
  • 换更大的 70 亿基座ALFWorld 96.1% vs 81.2%（高 15.0 分）——模型越大，增益不缩水；
  • 搬到没训过的新场景（ALFWorld Unseen，考"举一反三"）：86.2% vs 70.9%（高 15.3 分）——不是死记硬背，是真学到了可迁移的东西；
  • 省数据SEED 只用 60% 的训练数据，就追平 GRPO 用满数据的成绩；SEED 用 40% 数据 ≈ GRPO 用 80% 数据。又快又好，不用二选一。

最反直觉的两颗副发现（往往最值钱）：

副发现一：把技能"烤进权重"，胜过把同一条技能"贴进提示词"。作者做了个刁钻对照：另设一版 Skill-GRPO*，在考试时直接把技能塞进提示词给模型看（这相当于开卷、给了小抄）。结果——开卷的它，在 12 个综合指标里输了 11 个给 SEED。也就是说，同一条经验，临时贴在提示词上，比不过被真正训进模型身体里。这颗炸弹下文单独讲，它直接划定了"提示词的天花板"。

副发现二：技能必须"活着"现写，攒死的技能会拖后腿。消融实验里，把"每轮现写技能"换成"用一批固定的、事先写好的技能"，成绩掉了 7.4 分——是所有消融里掉得最狠的一处。印证：模型在变强，固定的小抄会迅速过时；"自我进化"这四个字不是修辞，是命门。

（其余消融：去掉第一幕的事后技能微调，掉 5.8 分；去掉第二幕的在线蒸馏，掉 4.8 分。两幕都不可省。）

7. 内核

一句话带走：当奖励只在终点给一个笼统的分时，别再干等外部批改——让模型回看自己刚走完的路、自造一份逐字的密集指点，并且把这份指点烤进权重、每轮重写。更锋利的那半句是：有些本事，塞进提示词只能"临时开卷"，只有训进权重才真正"长在身上"。

来龙去脉（倒读三层）

第一层：它站在谁肩上、又改了谁。SEED 直接接的是用强化学习训练 Agent 这条线，具体基线是 GRPO（分组相对策略优化，出自 DeepSeek-R1 那套配方，现在是 Agent 强化学习的默认起点）。它要修的病，就是 GRPO 的"奖励只在终点发、中间步骤没批改"。它用的工具是在线蒸馏（on-policy distillation，2026 年的大热门方向，已经有专门综述和 awesome 清单在追）——你上周读的 [[2026-07-15--direct-opd]]（Direct-OPD，弱到强的在线蒸馏）就是这条线，但那篇是"强老师教弱学生"，SEED 的新意恰恰是把外部老师去掉了：学生和老师是同一个模型的带/不带技能两副面孔，自己蒸馏自己。

第二层："事后总结技能"这个念头从哪来。它的祖上是两支——一支是 Hindsight Experience Replay（事后经验回放，2017 年强化学习经典：一局失败了别扔，换个角度看它"完成了另一个目标"，把废棋当成功经验回收）；另一支是会写反思笔记的 Agent，比如 Reflexion（2023，Agent 失败后用大白话自我批评一句、下次带着这句再战）、ExpeL、Voyager（给 Agent 攒一个技能库）。这些前辈都把"技能/反思"留在提示词里、而且攒死不更新。SEED 的两处进化就是针对这两个短板：（a）把技能从提示词搬进权重（用"带技能会把动作概率抬多少"这个差值当训练信号）；（b）把技能从攒死改成每轮现写（在线、贴着当前水平）。

第三层（根）：两条老河汇到这。一条是强化学习最古老的伤口——"功劳/过错分不清"：奖励只在终点给，中间每一步该记功还是记过，无从分配；从 1980 年代的时间差分学习一路到今天的 LLM Agent，都在跟它缠斗，SEED 是给它递上一份"自造的逐字批改"。另一条是这两年越来越粗的主干——自我进化 / 自我改进：不靠人标、不靠外部老师，让模型把自己的经验变成密集的自监督信号，自己喂自己长大。这条主干上，你台账里已经站着 [[2026-06-09--openskill]]（没有老师/答案/验题器时，Agent 自己造验证锚点练自己）、[[2026-07-06--automem]]（把"记忆"当成一种可学的认知技能）、[[2026-06-12--arbor-htr]]（存"洞见"而非"日志"的持久记忆）。SEED 是这条主干上，第一次把"自造的经验"直接变成逐字、进权重、还每轮刷新的监督信号。

对思染：模型进化 & 怎么用

这一步往哪进化了。前沿正在从"奖励结果"挪向"从模型自己的经验里，造出密集的监督信号"。训练流程正一块块被"会自造信号"的机制接管——这跟你昨天读的 [[2026-07-17--ring-zero-trillion-zero-rl]]（推理能力靠纯 RL 自己长出来，而非人写范例塞进去）是同一股风：越来越多的能力，不再靠人一条条喂进去，而是靠给模型一个能自动判对错、或能自我提炼的机制，让它自己长。 SEED 的独特贡献，是把这股风吹到了"密集监督"这块最缺的地上——它证明了模型回看自己刚走完的路，就能造出比稀疏奖励细得多的批改。

给提示词的那颗炸弹（最该记住的一条）。SEED 的副发现一，直接给"提示词工程"划了天花板：同一条经验，写进提示词里（哪怕考试时开卷直接塞给它看），也不如烤进权重里管用——开卷版在 12 个指标里输了 11 个。这对你意味着一个很硬的判断依据：

当你给京小贝的提示词里，规则、技巧、避坑清单越堆越长，它却还是不稳定地做不到那件事时——这往往不是"提示词没写够"，而是这件本事到了提示词的天花板，得往权重里搬了（微调 / 蒸馏），而不是继续加提示词。

打个你日常里铁定懂的比方：提示词是租来的房子，权重是买下的房子。便签能贴一屋子，但它始终是临时的、贴多了还互相打架；真要让一个习惯"长在身上"、稳定可靠，得把它砌进墙里。SEED 用实验量出了这堵墙的位置。

给京小贝产品认知的三条：

  1. 稀疏反馈是 Agent 产品的头号敌人——想办法造密集的中间信号。京小贝的多步流程（理解需求 → 查数据 → 判断 → 给方案），如果你只在"用户最后满不满意"这一个点上收反馈，中间哪步坏了永远查不出。SEED 的思路可搬：一段流程走完后，自动回看、提炼出"这次哪步是关键/哪步埋了雷"，把它变成下一次的密集指引——哪怕先只是喂回提示词或用于评测，也比只盯终点强。这也呼应你读过的 [[2026-06-21--fapo-autonomous-prompt-optimization]]（给多步管线做"失败归因"，定位是哪一步坏了）。
  1. 别信"攒死的小抄"——你的运营手册/提示词模板会随模型升级而过时。SEED 里"固定技能库"是掉分最狠的一处（−7.4）。同理：你今天为某个模型精心调的提示词技巧、话术清单，等你换了更强的基座或模型迭代了一版，很可能不再是最优、甚至变成累赘。手册要能贴着当前模型的行为重新长出来，而不是刻在石头上。
  1. "模型自己刚走完的一局"是最便宜的密集监督来源。不用额外标注、不用另训打分器——每一次完整的用户会话/Agent 轨迹跑完，都可以自动榨出"这次什么管用、什么要避开"再喂回去。对多用户、高频交互的京小贝，这是一口几乎白捡的井（但要注意跨用户的记忆治理边界——呼应 [[2026-06-23--gatemem-memory-governance]]，别把 A 用户的经验错喂给 B）。
