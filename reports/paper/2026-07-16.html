推荐理由

今天选 VP（Visual Pretraining，视觉预训练），因为它动的是这一整套技术里最底下、最没人怀疑过的那块砖。

大模型这几年的进步，全建立在一句没人明说的假设上：值得学的知识，都能无损地写成一串文字。 所以喂数据 = 把一切转成文本。PDF 论文、财报、网页，先用解析器压成纯文字，再切成 token 喂进去。这篇论文第一次正面问：这个"先转文字"的动作，是不是本身就在丢东西？

它的验证方式干净得可怕：同一批科学 PDF，同一个起点模型，同一套后续微调，只改一件事——一路把 PDF 解析成文字来训（TP），另一路直接把 PDF 页面渲染成图片来训（VP）。然后拿纯文字的推理考题去考。 结果是 VP 赢，而且只花了 25% 的 token 预算。也就是说：把论文当图看的模型，做纯文字数学题反而更强。

命中的质量信号：HuggingFace 本周（2026-W29）榜第 6、50 赞——在"核心能力机理"这条轴上是当周最高（票数在它上面的五篇是具身机器人和视频生成，不在这条轴上）；作者是上海人工智能实验室的书生（Intern）团队，署名里的陈恺、王文伟、郭麒麟是 InternLM 大模型和 XTuner 训练框架的作者群，这是有能力真跑百亿 token 级预训练的队伍，不是小作坊拍脑袋。另一个活信号：同一周 HuggingFace 上还有另外两篇独立撞进同一方向（《Let RGB Be the Language of Vision》《Video Generation Models are General-Purpose Vision Learners》）——三篇互不相干的队伍同时往"像素当基底"这个方向拱，说明这不是孤立巧思，是一条正在成形的真前沿。

对思染追前沿的价值：它给了一把可以立刻拿去用的尺子——"这页纸的信息，是在字里，还是在版式里？"（论文把它量化成了"视觉结构密度"，见结局那节的反直觉发现）。京小贝的输入天然是结构重的：持仓表、净值曲线、费率表、监管文件。"先 OCR 成文字再喂模型"这个默认动作，这篇论文说它是一次有损压缩，而且损失恰好集中在你最想让模型看懂的那类页面上。

诚实交底（三条，都重要）：

  1. 这篇不是"扔掉文本"。作者自己在局限那节写得很清楚：VP 是在已经文本预训练过的模型之上做续训，不是从零开始的独立视觉预训练。所以结论只能读成"视觉预训练能补充和扩展文本预训练"，不能读成"文本预训练可以不要了"。
  2. token 预算并非完全对等续训总量 TP 是 1800 亿 token、VP 是 1200 亿。作者解释是"同一批 PDF 在视觉流里本来就更紧凑"，不是换了数据源——但这仍是一处不完美的对照。
  3. 只在科学 PDF 这种高密度结构文档上验证过。能不能迁移到自然照片、视频，作者明说是开放问题。且看不到开源代码或模型权重。

七拍故事

1｜主角

台上站着两样东西。

一样是一页科学论文的 PDF——上面有一段文字、一个韦恩图（三个圆圈交叠、涂着阴影的那种）、一个排版好的公式、一张表格。

另一样是一句没人质疑过的话：模型要学这页纸上的知识，得先把它变成文字。

这个故事就是这两样东西的对决。

2｜困境

先把利害摆清楚。

现在训一个大模型学"科学知识"，标准动作是：拿一堆 PDF 论文 → 用解析器（论文用的是 MinerU 2.5，上海 AI Lab 自家的开源文档解析工具，把 PDF 里的字、公式、表格抽成结构化文本）压成纯文字 → 切成 token → 让模型做"预测下一个词"的填空练习。

问题出在第一步就把东西丢了。

那个韦恩图，解析器怎么处理？运气好，抽出一句 \[ A \cap (B \cup C) = (A \cap B) \cup (A \cap C) \]。可原图里那些三个圆圈怎么摆、哪块阴影压着哪块、哪个标签指着哪个区域——这些信息在图里是一眼就能看出来的关系，转成一行 LaTeX 之后，就只剩一串符号了。模型得自己在脑子里把这串符号重新画回三个圆圈，才能"看懂"。

论文引了一篇 1987 年的认知科学老文章来说这件事，标题就很直白：《为什么一张图有时值一万个词》（Larkin & Simon）。它的意思是：图之所以有用，不是因为它"好看"，而是因为图把某些关系直接摆在那里，让人不用推就能读出来；同样的关系写成文字，人得一步步推。

所以困境是：表格、公式排版、图文对应、整页的空间布局——这些东西里携带的信息，在"转成文字"这一步就被削掉或者削歪了。而没人真正测过，削掉的到底值多少。

利害在哪？在于数据墙。这两年所有人都在喊"高质量文本快用完了"。如果削掉的那部分其实很值钱，那"数据用完了"这个判断本身就是错的——不是数据没了，是我们一直用一种有损的读法在读同一批数据。

3｜旧路

之前的人不是没让模型看过文档图片。有两条老路，但两条都在同一个地方拐弯了。

第一条：免 OCR 的文档模型。 OCR（Optical Character Recognition，光学字符识别，就是"认图上的字"）。Donut（2021）、Pix2Struct（2023）、Nougat（2024）这一串工作，做的是"别先 OCR 了，直接把页面图片喂进去"。听着跟今天这篇一模一样？关键差别在目标：它们让模型看图，是为了吐出文字——把这页解析成 Markdown、回答这张表里的数字。图片只是入口，终点还是文字。本台账 06-24 那篇 Unlimited OCR / DeepSeek-OCR 这条线是这条路的最新一站，它的口号是"用图像当文字的廉价压缩"——像素是装文字的省钱容器，装完还是要还原成文字。

第二条：现在的多模态大模型。 InternVL、Qwen2-VL、LLaVA 这一类。它们训练时确实吃图片。但图片在损失函数里的身份是"条件"，不是"目标"。拆开说：模型的作业永远是"预测下一个文字 token"，图片只是摆在旁边帮它猜得准一点的参考资料。从来没有一道作业题的答案是图片本身。

论文一句话点破这两条路的共同毛病："原始文档被消费完就丢掉了，视觉这个模态始终待在模型的学习目标之外。"（In both regimes the raw documents are consumed and then discarded, leaving the visual modality outside the model's learning objective.）

翻译成大白话：你让模型看了图，但你从来没考过它图。 它没有任何压力去真正建模那个韦恩图的空间结构——因为考卷上不问这个。

4｜转折

作者转的这个弯，一句话就够：

别让图片当参考资料了，让图片本身当考题。

不解析成文字，不还原成文字，不拿文字当答案。就让模型预测"这页纸的下一块，长什么样"。

这个弯为什么之前没人转？因为有个技术上的硬钉子："预测下一个 token"这套玩法，靠的是答案是"离散"的——词表里就那 15 万个词，模型输出一个概率分布，说"下一个词是'的'的概率 0.3"，答案对不对一目了然。

可一块图像的特征是一个连续的向量（几百个小数，不属于任何词表）。你怎么让模型"预测"一个连续向量？没有词表，就没法算那个概率分布，标准的那套损失函数直接失效。

这就是转折的真正含金量：他们找到了在连续空间里做自回归填空的办法（下一拍说清）。

5｜解法

VP 的完整机制，拆成五步，每步都说清为什么这么设计。

第一步：把页面变成一串图块。 一页 PDF 渲染成图片 → 送进一个冻住的视觉编码器（vision tower，就是个已经训练好的看图模型，这里把它参数锁死不再改）→ 输出一串特征向量 z₁…z_N。每个向量对应页面上的一小块（patch，比如 14×14 个像素的一个小方格）。

为什么把视觉编码器冻住？ 因为它要当"标准答案的出题人"。如果它一边出题一边自己也在变，模型就可以作弊——把所有图块的特征都塌缩成同一个值，那预测起来当然百发百中，但什么也没学到。冻住它，标准答案才是稳的。

第二步：把空白扔掉。 一页纸绝大部分是白的——页边距、行间距、空白。作者用最土的办法筛：算每个小方格的像素方差和平均亮度（方差 = 这块里深浅变化大不大；全白的方格方差接近 0）→ 方差小又亮的，判定为背景，直接删。

留下的图块按从左到右、从上到下（raster scan，就是老式电视扫描的顺序，也就是人读纸的顺序）排成一串 u₁…u_L。论文里写 L ≪ N（远小于）——大部分格子都被扔了。

为什么这一步很关键？ 因为它是"便宜"的来源。同一批 PDF：解析成文字要 800 亿 token，渲染成图只要 200 亿 视觉 token。图片版比文字版还省 4 倍。 这反直觉——图不是应该比字占地方吗？答案是：一页纸的字里行间和页边距，占了大部分面积却不携带信息，扔掉之后，剩下的"有东西的部分"反而比把每个字拆成 token 更紧凑。

第三步：塞进同一个大脑。 每个留下的图块特征，用一个线性投影（就是乘一个矩阵，把向量从"视觉编码器的语言"翻译成"大模型内部的语言"，维度对上）送进 LLM。位置编号按刚才的扫描顺序重新排——这样版面的空间关系就被保住了：模型知道这块在那块的右边、下一行。

注意：送进的是同一个自回归主干，就是那个平时处理文字 token 的 Transformer，没有另开一个视觉分支。用的还是因果掩码（causal mask，看得见前面、看不见后面，跟写文章一个字一个字往下写一样）。

第四步：考它下一块长什么样（核心）。 模型在位置 t 吐出一个隐状态 → 一个输出投影头把它映射回"冻住的视觉编码器那个特征空间" → 得到预测 ẑ_{t+1} → 拿它跟真正的下一块 z_{t+1} 比。

怎么比？用对比损失，同批次的其他图块当反例（InfoNCE，出自 2018 年 van den Oord 的对比预测编码）。大白话拆开：

一个批次里有一堆图块。模型预测出一个向量，规则是：你这个预测，得离"真正的下一块"最近，同时离"这批里其他所有图块"都远。 离对的近、离错的远，就得分。

公式是 L_VP = -(1/|B|) Σ log p_ii，其中 p_ii 就是"我猜的这个，正好匹配上真正那个"的概率。

为什么用对比损失而不是直接算距离？ 因为如果只要求"离真答案近"，模型有个偷懒解：永远输出所有图块的平均值——反正离谁都不太远，损失还挺低。加上"必须离别的图块远"这个要求，偷懒解立刻失效：平均值离每个图块都一样近，就分不出你猜的是哪一块。反例的存在，逼模型必须真的分辨"这一块"和"那一块"的区别。

第五步：文字和图一起练。 总损失 L = λ_text · L_CE + λ_vis · L_VP。前半是老的文字填空（CE = 交叉熵），后半是新的视觉填空，按固定比例交替喂。

训的时候：改 LLM 主干 + 视觉输入投影 + 预测头；不改 视觉编码器（还是那句：它得当稳定的出题人）。用 XTuner 框架（书生自家的训练工具）跑。

这一步暴露了作者的诚实不是"扔掉文字只喂图"，是两样一起喂。所以严格说，VP 是给现有预训练加了一门新课，不是换了个学校。

6｜结局

主结果（表 1）：把论文当图看的模型，做纯文字题更强。

对照极其干净：同一个起点模型、同一批 PDF、同一套 SFT（监督微调）、同样的考卷，唯一的差别是那批 PDF 是"解析成文字"还是"渲染成图"。而且考的是纯文字的推理题——考卷上一张图都没有。

注意 Qwen 3.5 那行 GPQA：文字续训把成绩从 77.84 干到了 76.24——喂了 800 亿 token 的解析文本，模型反而变差了。 而同一批文档换成图喂，涨到 79.29。

效率：VP 只用了 25% 的 token 预算。 200 亿视觉 token vs 800 亿文字 token，还赢。

scaling（图 2b）：优势随训练量拉大，不是一次性小便宜。 把 VP 相对 TP 的增益归一化后，随训练推进稳步爬到 MMLU-Pro 1.27×、GPQA 2.02×、AIME-25 2.88×。

接下来是三个反直觉的副发现，最有意思的部分。

反直觉一：好处只出现在"结构重"的页面上——这恰恰是这篇最该被信的理由。

作者按视觉结构密度（页面上图、表、公式、版式占多少）把考题分三档，看 VP 的增益：

视觉结构密度VP 相对文字基线的增益
低（几乎纯文字的页）−0.15（不涨，还略跌）
中+2.86
高（图表公式版式密集）+3.80

这条为什么重要？ 因为它是一次主动的证伪测试。如果 VP 只是"多喂了点算力所以普涨"，那低密度的页也该涨。可它偏偏在纯文字页上不涨甚至略跌——这说明涨的那部分确确实实来自版式里的信息，不是训练玄学。作者自己也这么写："这证实 VP 的收益是真正由建模视觉内容驱动的。"

这条也是今天最能直接搬走的东西（见下面"对思染"）。

反直觉二：视觉 token 给得越多，并不越好。

先涨后跌。作者的解释很诚实——怀疑是 InfoNCE 那个对比损失自己的毛病：留的图块越多，一个批次里的"反例"就越多，损失的有效梯度尺度就变了，而学习率这些超参在这组扫描里是固定没调的。换句话说，这可能是调参没跟上，而不是"信息多了有害"的深刻规律。作者没有硬把它讲成规律。

顺带一条同方向的：把渲染分辨率从 1120 降到 560，性能基本保住，还是赢 TP。图还能再压。

反直觉三：文字续训在看图这件事上，会把模型练坏。

多模态考卷上（表 2b），Qwen 3.5 的 ChartQAPro（读图表回答问题）：

  • 起点 Base：57.99
  • 文字续训 TP：56.42（掉了 1.57）
  • 视觉续训 VP：61.80（涨了 3.81）

给模型灌更多"从图表里解析出来的文字"，让它读图表的能力变差了。 这不是小概率噪声——作者把它当作核心证据之一："这表明继续做文字预训练，已经没法从视觉密集的文档里再榨出信息了。"

还有一个白捡的好处：模态鸿沟自己合上了。

一直有个known问题叫模态鸿沟（modality gap，2022 年 NeurIPS 那篇《Mind the Gap》命名的）：模型脑子里"一张猫的图"和"'猫'这个字"的表示，明明指同一件事，却飘在两个互不重叠的区域里。

互为最近邻的比例 @10.1400.310

关键在于：整个训练过程里，没有喂过一对"这张图配这段文字"的标注。 图文对齐是白捡的副产品——你只是逼模型好好预测下一块图，它就顺手把图和字放到同一个空间里了。

论文还做了个注意力探针（图 4）：同一道数学题，一路给文字、一路给图片。结果两边的注意力落在同一片语义区域上——题目的约束条件、中间计算步骤，都被高亮；填充性的废话都被冷落。作者在这里也很克制："这不意味着两个模态用的是同一套内部机制，但它显示视觉通路确实把推理相关的证据暴露给了共享的 LLM。"

7｜内核

文字不是知识本身，是知识的一次投影。投影会丢东西——而丢掉的那部分，可以直接从像素里学回来，还更便宜。

"先转成文字"从来不是一个中立的预处理步骤。它是一次有损压缩。而这篇论文最狠的地方在于它量化了损失落在哪：页面结构越重，损失越大（+3.80 vs −0.15）。

来龙去脉（倒读三层）

第一层：它在改进谁？——"读像素"这条路，之前一直在为"吐文字"服务

直接前身是免 OCR 的文档模型这一串：Donut（2021，Naver）→ Pix2Struct（2023，Google）→ Nougat（2024，Meta）。它们早就在干"别先 OCR，直接吃页面图"。

但它们的终点全是文字。 Donut 的全名就是"没有 OCR 的文档理解 Transformer"，目标是解析、理解、图转文。论文原话点破："它们的预训练主要是为解析、理解、或者图到文的生成而优化的。"

本台账 06-24 的 Unlimited OCR / DeepSeek-OCR 那条线，是这条路目前跑得最远的一站，它的核心洞见是"用图像当文字的廉价压缩"——一页文字渲染成图，比切成 token 更省，10 倍压缩还能还原 97%。

看清这个分岔，就看清了今天这篇的位置：

  • 06-24 那条线说：像素是装文字的更省钱的容器。（装进去，最后还是要倒出来变成文字）
  • 今天这篇说：像素里装着文字从来就没有的东西。（不倒出来了，损失函数直接长在视觉空间里）

同样是"把纸当图看"，一个是压缩，一个是基底。这是两个完全不同的野心。

第二层：它借了什么零件？——怎么在"连续"的世界里做填空

转折那拍留了个钉子：答案是连续向量时，"预测下一个"怎么算分？作者是从两个地方拿的零件：

零件一：InfoNCE / 对比预测编码（van den Oord，2018）。提供了"用同批次里别的样本当反例"这个基本套路——不要求你说出正确答案的绝对坐标，只要求你把对的排在错的前面。 这是 VP 损失函数的直接来源。

零件二：MAR（Autoregressive image generation without vector quantization，Kaiming He 组，NeurIPS 2024）。这篇证明了一件当时反直觉的事：做图像自回归，不必先把图量化成离散码本（以前的标准做法是先训个"图像词表"把每块图映射成一个整数，才能套用文字那套 softmax）。连续的目标也能做自回归。

VP 干的事，是把这两个零件焊到一个语言模型主干上，并且让它跟文字损失共存（λ_text·L_CE + λ_vis·L_VP）。所以 VP 的新意不在发明了什么全新数学，而在接线的位置：第一次把"预测下一块图"这道题，接到了那个平时预测下一个词的同一个大脑上，并且证明这么练完，文字能力也涨了。

第三层（根）：被攻击的那条根假设

再往下挖，这篇真正顶着的是Kaplan 缩放定律（2020）、GPT-3（2020）、Chinchilla（2022） 这一整条主干路线的一个隐含前提。论文引言第一段就把它拎出来了，而且用词很重——"这套范式建立在一个很强的隐含假设上：所有值得学的知识，都能被无损地编码成一串线性的文本 token。"

注意"线性"这个词。文字是一维的——一个字接一个字。而一张韦恩图是二维的，一个表格是二维的。把二维的东西压成一维，天然要丢掉"谁在谁旁边"这类关系。 这个前提从来没被系统检验过，因为它太自然了：大家都在文本上训，效果好，那就继续。

给这次攻击提供理论弹药的，是柏拉图表征假说（Platonic Representation Hypothesis，Huh、Cheung、Wang、Isola，2024）。拆开说：这个假说观察到，不同的模型、用不同的模态训出来的表征，会慢慢收敛到同一套关于现实的抽象上——就像柏拉图那个山洞比喻，大家看到的是同一个真实物体投在墙上的不同影子。

如果这个假说成立，那么推论是致命的：从一篇科学论文里抽出来的文字，只是那个更丰富的底层结构的一次投影。而这个结构，原则上可以直接从原始的视觉文档里学到。 论文原话就是这么写的。图 1 右边那两张 t-SNE 图（把高维表示压到三维画出来看的可视化）就是在给这句话上视觉证据：训练前，文字表示（灰）和视觉表示（蓝）是两堆分开的云；VP 训完，两堆云塌进同一个区域。同一个底层对象的两个影子，终于被放回了一起。

再往下就是认知科学的老地基Larkin & Simon 1987《为什么一张图有时值一万个词》、Barsalou 的知觉符号系统（1999）和具身认知（2008）、Zhang 1997 关于外部表征在解题中的作用。这些几十年前的工作早就说过：人类之所以画图，不是为了好看，是因为图让某些推理"不用推就能看出来"。 这篇论文，本质上是把这个几十年前的认知科学结论，第一次做成了大模型预训练的损失函数。

对思染：模型进化 & 怎么用

一、这对"模型进化逻辑"意味着什么

前沿在换基底。

过去两年所有关于"数据墙"的焦虑——高质量文本快用完了、合成数据会不会模型自噬——全都默认了一件事：数据 = 文本。 这篇给了一个很不一样的答案：

同一批文档，换一种读法，还能再榨出信息来，而且 token 还便宜 4 倍。

如果这条成立，"数据用完了"这个判断本身就得改写：不是数据没了，是我们一直用一种有损的读法在读同一批数据。 世界上所有的 PDF、幻灯片、财报、教科书、网页截图——都被"先转文字"这道工序削过一遍，削掉的部分还在原地躺着，没人捡。

再往前看一步，作者在展望里说得很直白，这是最值得记住的一句：

未来的基座模型，可能主要在大规模视觉流上训练，只配一层轻量的文本对齐。

这跟 Karpathy 一直念叨的"删掉 tokenizer"是同一个方向上的两块拼图——tokenizer（把文字切成 token 的那个部件）是人类硬塞给模型的一层中间商，而像素是更接近原始信号的基底。 这篇是第一次有人拿百亿 token 级的对照实验，给这个方向提供了正面证据。

但要跟住边界，别把它读成结论（这三条我在前面也标了，值得再钉一次）：

  • 作者自己写了 VP 不是独立于语言预训练的视觉预训练——它是在文本预训练好的模型上做续训。这是"加一门新课"，不是"换个学校"。
  • 只在科学 PDF 这种高结构密度文档上验过。自然照片、视频，作者明说是开放问题。
  • HLE（最难那套考卷）只涨了最多 0.97 分，作者的解释很关键：HLE 难在硬碰硬的多步推理，不难在知识。 所以 VP 补的是知识，不是推理力。这条边界很干净：版式里藏的是知识，不是智商。

二、能怎么用

落到提示词：多了一把叫"结构密度"的尺子

这是今天最能立刻搬走的东西。那张三档表（低 −0.15 / 中 +2.86 / 高 +3.80）不只是论文里的一个消融实验，它是一把判断"什么时候不该转文字"的尺子：

喂材料给模型之前，先问一句：这页的信息，在字里，还是在版式里？

  • 信息在字里（纯文字的政策解读、新闻、访谈记录）→ 转文本无所谓，论文实测低密度页 VP 优势是 −0.15，也就是没优势。复制粘贴就行。
  • 信息在版式里（持仓表、费率对比表、净值曲线图、财报三张表、带脚注的监管文件）→ 别转。直接把页面图喂给多模态模型。 论文实测高密度页差 +3.80。

他自己平时用 Claude/GPT 读研报、读财报的时候，这条今天就能用上：遇到表格和图表密集的 PDF，别复制文字进去，直接截图或者传原 PDF。 以前这么做只是"感觉图更全"，现在有了量化的理由和边界——而且知道了它在纯文字页上不管用，不用瞎截图。

落到京小贝：两个可测的动作 + 一个警告

京小贝的输入天然是结构最重的那一类：基金定期报告的持仓表、净值曲线、费率表、银行流水、监管文件。现有链路大概率是标准动作——OCR / 解析成文本 → 喂模型。

动作一：对结构重的页做一次 A/B。 挑一批表格和图表密集的材料（持仓表、费率对比），一路走现有的"解析成文本"链路，一路直接把页面图喂多模态模型，专测那些答案藏在表格结构和图里的问题。这篇论文预测的差距是 ChartQAPro 上 +5.4 分级别——如果京小贝的链路里真有这个缺口，这个测法一天就能看出来。

动作二：把"结构密度"变成路由条件。 不用一刀切。低密度的纯文字页继续走解析（便宜、快、论文说没损失）；高密度的结构页走视觉通路。这是论文那张三档表能直接变成的工程规则。

警告（这条最该记）： ChartQAPro 上文字续训掉分那个发现（57.99 → 56.42），是个很具体的反直觉信号：

给模型灌更多"从图表解析出来的文字"，可能让它读图表的能力更差。

这跟直觉完全相反——通常我们觉得"喂更多领域文本 = 模型更懂这个领域"。但如果这批文本是从视觉密集的材料里解析出来的，你灌进去的是这些材料的残骸，而模型会照着残骸调整自己。

跟 06-24 那篇连起来看，这一层就完整了：

  • 06-24 提醒的是你的解析器准不准？（编辑距离、跨页丢没丢）
  • 今天这篇提醒的是更狠的一层解析器再准，它准的也只是"文字部分"。版式里那些信息，根本不在它的输出空间里——它不是没解析对，是它压根就没有那个输出档位。

所以"换个更强的 OCR"解决不了这个问题。这是表征选择的问题，不是解析质量的问题。

一句可执行的判断

别再默认"先转文本"是一次无损的预处理。它是一次有损压缩，而且损失恰好集中在你最想让模型看懂的那类页面上。
